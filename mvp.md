# Purpose

This document defines the Minimum Viable Product (MVP) for ARGOS.

The MVP represents the smallest version of ARGOS that delivers meaningful value while remaining faithful to the project's architecture and philosophy.

The purpose of the MVP is to establish a focused implementation target, prevent feature creep, and provide a clear path toward the first functional release.

---

# MVP Vision

The first usable version of ARGOS should become a tool that Arjun chooses to use every day.

Success is measured by usefulness, reliability, and maintainability—not by the number of implemented features.

---

# MVP Goals

The MVP should demonstrate the core ideas behind ARGOS:

* Human-first interaction
* Cognitive reasoning
* Memory with user consent
* Modular architecture
* AI model independence
* Practical computer automation

---

# Included Features

## 1. User Interaction

ARGOS shall support:

* Wake word activation ("Argos")
* Voice conversations
* Text conversations
* Natural language understanding

---

## 2. Brain Core

The Cognitive Engine (Brain Core) shall:

* Understand user intent
* Plan tasks
* Route requests
* Reflect on completed tasks
* Generate responses

---

## 3. Memory System

The MVP includes:

* Working Memory
* Session Memory
* Long-Term Memory
* Memory Consent

The following are excluded from the MVP:

* Healthy Forgetting
* Memory Ranking
* Automatic Knowledge Extraction

---

## 4. Model Router

ARGOS shall:

* Support multiple AI models
* Route tasks to appropriate models
* Support local model fallback
* Respect user-defined policies

---

## 5. Computer Control

ARGOS shall be capable of:

* Opening applications
* Opening websites
* Reading files
* Creating folders
* Searching files
* Running terminal commands (approved only)

---

## 6. Productivity

The MVP should assist with:

* Coding support
* Documentation writing
* PDF summarization
* Note generation
* Basic reminders

---

## 7. Policies

The MVP includes:

* User confirmation for sensitive actions
* Offline behavior policies
* Model selection policies
* Privacy rules

---

## 8. Explainability

ARGOS should be capable of answering questions such as:

* "Why did you do that?"
* "Which model did you use?"
* "What are you doing?"
* "What are you waiting for?"

Transparency is considered a core feature rather than an optional enhancement.

---

# Explicitly Excluded

The following features are intentionally excluded from the MVP:

* Emotion recognition
* Facial recognition
* Smart home integration
* Robotics
* IoT device control
* Mobile application
* Browser extension
* Autonomous self-improvement
* Multi-user support
* Plugin marketplace

These features may be considered in future releases after the core platform is stable.

---

# Success Criteria

The MVP is considered successful when:

* ARGOS can understand natural language requests.
* ARGOS can perform common computer tasks reliably.
* ARGOS remembers approved user preferences.
* ARGOS routes requests between AI models appropriately.
* ARGOS follows the Constitution and Engineering Decisions.
* Arjun chooses to use ARGOS during normal daily work.

---

# Non-Goals

The MVP is **not** intended to:

* Replace professional software.
* Be fully autonomous.
* Match the capabilities of fictional AI assistants.
* Implement every planned subsystem.

The objective is to validate the architecture through a useful, maintainable implementation.

---

# Engineering Rule

Any proposed feature must answer the following question before being added to the MVP:

> **Does this feature significantly improve the usefulness of the first release?**

If the answer is **No**, the feature shall be deferred to a future version.

---

# Release Target

This MVP is planned for:

**Version 0.4.0 — Awakening**

Completion of the MVP marks the transition from architectural planning to a usable Cognitive Operating System prototype.

---

# Summary

The MVP is intentionally limited in scope.

Its purpose is not to build everything at once, but to prove the architecture, establish reliability, and create a system that provides real value in everyday use.

The MVP serves as the foundation upon which all future ARGOS capabilities will be built.
